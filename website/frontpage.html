Why the SourceForge project name wwwsearch, and not mechanize? When I created the SF project, the immediate problem I wanted to solve was performing searches on a database website with an unpleasant user interface, and the plan was to write code to help specifically with that kind of task. Later, I shifted attention to general web scraping tasks.
